What this project is:
- A playful Python helper/library that exposes common Python built-ins with Thai names.
- The goal is not production readiness yet; it is currently a prototype / playground for Thai-first coding syntax.
- The project intentionally explores whether Python code can feel more natural to Thai speakers by replacing common built-in names with Thai vocabulary.

Current file structure:
- ไพท่อน.py
  - Main library version.
  - Uses simple alias style like `พิมพ์ = print`.
  - This is the preferred main file right now.
- ไพท่อน_builtins.py
  - Alternate library version.
  - Uses explicit import style like `from builtins import print as พิมพ์`.
  - Kept mainly for comparison and design discussion.
- x.py
  - Minimal example: input + output.
- y.py
  - Slightly richer example using type conversion, length, sorting, max, all.
- README.md
  - Current human-readable project summary and next-step notes.

Current behavior choices:
- เรียงลำดับ(...)
  - Intentionally behaves like list.sort()
  - Mutates the original list and returns the list
- เรียงแล้ว(...)
  - Intentionally behaves like sorted()
  - Returns a new sorted value

Important technical caveat:
- There is a Unicode normalization issue with some Thai identifiers.
- Specifically, words like ลำ and จำนวน can be typed in visually similar but different Unicode forms.
- This caused the static analyzer to report names as undefined even when they looked correct to the eye.
- To stabilize the prototype, compatibility aliases were added for alternate Unicode forms.
- Examples currently included:
  - เรียงลําดับ = เรียงลำดับ
  - จํานวนจริง = จำนวนจริง
  - จํานวนเต็ม = จำนวนเต็ม
- If future work introduces more Thai identifiers with sara am or similar characters, expect the same issue and handle carefully.

Current compatibility stance:
- The new Thai-native names are intended to be the main API.
- Some older transliterated names are still kept as compatibility aliases inside the library files so older examples do not immediately break.
- A future cleanup pass may remove those compatibility aliases if the project decides to go fully Thai-only.
